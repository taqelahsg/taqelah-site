Prompt Evaluator Setup Guide - Complete Installation Tutorial

December 4, 2025 15 min read ... reads Tutorial

Syam Sasi

Co-Founder, taqelah!

Traditional tooling for machine-learning workflows does not address the unique quality-assurance needs of modern LLM applications. LLM systems require far more than numeric metrics ‚Äî they need semantic correctness checks, hallucination detection, RAG grounding evaluation, security and injection testing, bias auditing, multi-provider comparison, and token-level cost tracking. Existing tools rarely capture these dimensions, leaving teams blind to subtle output regressions, safety vulnerabilities, and unexpected billing spikes. Without these capabilities, it becomes difficult to ensure that prompts behave reliably, consistently, and securely across different models and datasets.

Prompt Evaluator fills this gap by providing a dedicated, test-driven framework for validating and benchmarking prompts at scale. It offers assertion-based testing, multi-provider evaluations, OWASP LLM Top-10 security checks, drift detection, and detailed cost/latency tracking ‚Äî all wrapped in an intuitive desktop GUI that non-technical stakeholders can use without writing code. Built on the flexible Promptfoo engine, it supports local execution, CI/CD integration, extensible evaluators, and privacy-first workflows. Compared to research-focused or cloud-locked alternatives, Prompt Evaluator delivers a complete, enterprise-ready QA layer that ensures prompts are reliable, safe, and cost-efficient before they reach production.

üß© Step 1 ‚Äì Install Node.js (Required)

Node.js is the foundation that powers Promptfoo, which Prompt Evaluator depends on.

  1. Go to https://nodejs.org/en
  2. Download the LTS version for your OS (Windows / macOS)
  3. Run the installer and finish the setup
  4. Confirm it works by opening your terminal and running:
node -v
npm -v

You should see version numbers for both commands, confirming Node.js is installed correctly.

Screenshot: Node.js version check

üì¶ Step 2 ‚Äì Install Promptfoo (CLI)

Prompt Evaluator expects Promptfoo to already be installed on your system.

  1. Open Terminal (macOS) or Command Prompt / PowerShell (Windows)
  2. Install Promptfoo globally using npm:
npm install -g promptfoo
  1. Verify the installation:
promptfoo --version

Once this command shows the version number, you're ready to install the desktop app.

Screenshot: Promptfoo version check

üíª Step 3 ‚Äì Install Prompt Evaluator Desktop App

Download the App

  1. Go to the official releases page: https://github.com/syamsasi99/prompt-evaluator/releases/tag/v0.0.1
  2. Download the appropriate installer for your OS:
    • macOS: Download the .dmg file
    • Windows: Download the .exe file
    • Ubuntu/Linux: Download the .AppImage or .deb file

Installation & Opening Self-Signed Apps

Since this app is self-signed, you'll need to follow these steps to open it on your operating system:

‚ö†Ô∏è Note: Prompt Evaluator is self-signed, which means it hasn't been notarized by Apple/Microsoft. This is normal for open-source desktop applications. The steps below will help you safely open the app.

For macOS Users:

  1. Open the downloaded .dmg file and drag Prompt Evaluator to your Applications folder
  2. When you try to open it the first time, you'll see a warning: "Prompt Evaluator cannot be opened because it is from an unidentified developer"
  3. To bypass this:
    • Option 1 (Recommended): Right-click (or Control-click) on the app ‚Üí Select Open ‚Üí Click Open in the dialog
    • Option 2: Go to System Settings ‚Üí Privacy & Security ‚Üí Scroll down to find "Prompt Evaluator was blocked" ‚Üí Click Open Anyway
  4. On first launch you'll see a "Setting up Prompt Evaluator" screen
  5. It will check for Promptfoo and other dependencies
  6. When you see "All dependencies verified successfully!" ‚Üí click Continue

Screenshot: macOS warning

Screenshot: Open Anyway

For Windows Users:

  1. Run the downloaded .exe installer
  2. If you see "Windows protected your PC" warning:
    • Click More info
    • Then click Run anyway
  3. Follow the installation wizard to complete the setup
  4. Launch the app from Start Menu or Desktop shortcut
  5. On first launch you'll see a "Setting up Prompt Evaluator" screen
  6. It will check for Promptfoo and other dependencies
  7. When you see "All dependencies verified successfully!" ‚Üí click Continue

For Ubuntu/Linux Users:

Using AppImage (Recommended):

  1. Download the .AppImage file
  2. Make it executable:
chmod +x prompt-evaluator-*.AppImage
  1. Run the AppImage:
./prompt-evaluator-*.AppImage

Using .deb package:

  1. Download the .deb file
  2. Install it:
sudo dpkg -i prompt-evaluator-*.deb
sudo apt-get install -f  # Fix any dependency issues
  1. Launch from Applications menu or run prompt-evaluator in terminal
  2. On first launch you'll see a "Setting up Prompt Evaluator" screen
  3. It will check for Promptfoo and other dependencies
  4. When you see "All dependencies verified successfully!" ‚Üí click Continue

üóÇ Step 4 ‚Äì Create Your First Project

  1. On the top-left, click the project dropdown
  2. Use the default "My First Project" or create a new one

üí¨ Step 5 ‚Äì Add a Prompt

  1. In the left sidebar, click Prompts
  2. Create a sample system/user prompt, for example:
You are a helpful assistant. Please answer the following question in JSON format:
{"answer": "<your answer here>"}

Use a variable like {{question}} if you plan to use a dataset column later.

Screenshot: Prompt editor

üîê Step 6 ‚Äì Configure Model Providers (OpenAI, Gemini, etc.)

  1. Click Providers in the left sidebar
  2. Click + Add Provider

For OpenAI:

  • Set Provider ID to something like openai:gpt-4o
  • Paste your OpenAI API key in the API Key (Optional) field
  • Click Test ‚Üí wait for "API Key Validated Successfully"
  • Choose OK if you want it saved into your .env file

For Google Gemini:

  • Add another provider with ID like: google:gemini-2.5-flash-lite (or any supported model)
  • Paste your Gemini API key
  • Click Test ‚Üí validate ‚Üí save to .env if you like

üí° Pro Tip: You can keep up to 10 providers. The app will compare them on the same dataset and assertions.

üìä Step 7 ‚Äì Create a Dataset

  1. Click Dataset in the sidebar
  2. You have three options:

Option A ‚Äì Paste Table Data

Paste a simple CSV-like block:

Click Parse & Load.

Option B ‚Äì Generate with AI

  • Click Generate with AI (top-right in Dataset view)
  • The app will auto-generate test rows based on your prompt variables

Option C ‚Äì Upload CSV

Use Choose File to upload a CSV with headers matching your variables (e.g., question).

(Optional) Click Generate Expected Output to let AI create an expected_output field (e.g. {"answer": "Jupiter"}) for factuality checks.

üîí Step 16 ‚Äì Security Testing with OWASP LLM Top 10

Prompt Evaluator includes built-in security testing capabilities to help you identify vulnerabilities in your LLM applications. The tool integrates OWASP LLM Top 10 security checks that can be easily enabled from the settings page.

Enable OWASP LLM Security Tests

  • Go to Settings ‚Üí Security
  • Enable the OWASP LLM Top 10 security tests
  • These tests will automatically check for common LLM vulnerabilities including:
    • Prompt injection attacks
    • Insecure output handling
    • Training data poisoning
    • Model denial of service
    • Supply chain vulnerabilities
    • Sensitive information disclosure
    • Insecure plugin design
    • Excessive agency
    • Overreliance on LLM outputs
    • Model theft

Running these security tests helps ensure your prompts are robust against common attack vectors and potential misuse before deploying to production.
